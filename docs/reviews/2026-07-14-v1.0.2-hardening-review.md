# v1.0.2 Hardening Review

## Scope

Review target: `codex-dmx-proxy` at the v1.0.2 hardening candidate. Review
covered every tracked runtime, installer, platform-adapter, control, test,
metadata, CI, and documentation file.

## Findings and Disposition

| Finding | Risk | Disposition | Evidence |
|---|---|---|---|
| Prefix-only `http(s)` test replayed malformed URL values | Whole conversation rejected by strict third-party Responses validation | Fixed | `_is_replayable_remote_image_url` uses structured parsing, host, port, and whitespace checks; malformed regression cases pass. |

## Security and Data Boundaries

- The service remains loopback-only by default and uses a direct `urllib` opener
  that bypasses system proxy interception for upstream traffic.
- The proxy keeps Bearer credentials in transit only. Request-body dumps are
  opt-in diagnostic behavior and are ignored by Git.
- Route state does not store configuration text or secrets; it stores only file
  paths, route identity, and SHA-256 values. The backup remains local and is
  checked before restoration.
- The proxy transforms only outbound `/responses` payloads. It does not mutate
  Codex sessions, SQLite state, archives, or transcript history.

## Verification

- Python 3.12, 3.13, and 3.14 compile and unit-test matrix: passing locally.
- Package test suite: 30 test cases covering sanitized images, encrypted replay,
  Windows launcher generation, install-state activation, `CODEX_HOME`, safe
  uninstall, and metadata identity.
- Release metadata validator: passing for `1.0.2` before tagging; tag verification
  is rerun after annotated `v1.0.2` creation.
- Deployment proof is performed after publication by comparing the installed
  payload with this release, then exercising the active loopback proxy and an
  authenticated Codex Responses request.

## Limits

- Windows Scheduled Task and Linux service adapters have structural and generated
  artifact coverage here; this macOS workstation cannot provide a physical
  Windows or Linux service-manager acceptance run.
- No software release can prove the absence of all future defects. This release
  makes its covered behavior and remaining environment limits explicit.
